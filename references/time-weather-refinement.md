# 网页端时间/天气细化流程

## 一、为什么需要细化

**MCP 工具的局限：**

| 字段 | MCP 提供 | 网页端提供 |
|------|---------|-----------|
| 训练日期 | ✅ 日期（无时分秒） | ✅ 精确到分钟 |
| 上午/下午/晚上 | ❌ 无法区分 | ✅ "晚上 08:21" |
| 一天多次训练 | ❌ 日期重复，无法区分 | ✅ 精确时间可区分 |
| 时段天气 | ❌ 需第三方 API | ✅ 页面自带训练时的天气 |

**典型场景：**
- "昨晚的间歇跑和今早的轻松跑"——MCP 无法区分早晚
- "跑步时实际温度/湿度"——网页端直接给出，无需猜 API
- "一天跑了两次，哪次是晨跑哪次是夜跑"——精确时间可区分

---

## 二、细化流程

```
① 分析触发 → 判断是否需要时间/天气精度
↓
② 需要精度？
   ├── 是 → 检查网页端登录态（hasLoginState()）
   │         ├── 有 → 走网页端 fetchActivityDetail 获取 startTime + weather
   │         └── 无 → 询问用户："是否需要细化数据（精确时间和天气）？"
   │                    ├── 是 → 提示运行 coros_web_login.js → 授权后重试
   │                    └── 否 → 降级到 MCP 近似数据
   └── 否 → 继续 MCP 标准流程
```

## 三、调用方式

```javascript
const { fetchActivityDetail, hasLoginState } = require('{workspace}/高驰数据分析/coros_detail.js');

// 1. 检查是否有登录态
if (hasLoginState()) {
  // 2. 有 → 直接抓取
  const data = await fetchActivityDetail(labelId, sportType);
  const { startTime, weather } = data;

  // startTime: { date, period: "晚上", time: "08:21" }
  // weather:  { condition: "小雨", temp: 31, humidity: 76, wind: "东风 6" }
} else {
  // 3. 无 → 询问用户
  console.log('⚠️  需要细化数据（精确时间和天气），是否授权网页端？');
  console.log('   如需授权，运行: node coros_web_login.js');
  // 等待用户响应… 如果同意则引导授权
}
```

## 四、返回值说明

### startTime（精确开始时间）

```json
{
  "raw": "2026年6月4日 晚上 08:21",
  "date": "2026-06-04",
  "period": "晚上",
  "time": "08:21"
}
```

**period 取值：** `"早晨"` / `"上午"` / `"下午"` / `"晚上"` / `""`（未知）

**判定规则（用于分析）：**
| period | 可判定 | 说明 |
|--------|--------|------|
| 早晨 | 晨跑 | 5:00-8:00 |
| 上午 | 上午跑 | 8:00-12:00 |
| 下午 | 下午跑 | 12:00-18:00 |
| 晚上 | 夜跑 | 18:00-24:00 |
| 无 period（仅有时间） | 按 HH 判断 | — |

### weather（训练时段天气）

```json
{
  "condition": "小雨",
  "temp": 31,
  "humidity": 76,
  "wind": "东风 6"
}
```

| 字段 | 意义 | 说明 |
|------|------|------|
| condition | 天气现象 | 小雨、多云、晴 等 |
| temp | 温度（℃） | 训练时实际温度 |
| humidity | 相对湿度（%） | 训练时实际湿度 |
| wind | 风力和风向 | 如 "东风 6" |

> 网页端天气数据**优先**于 OpenWeather/Open-Meteo API，因为它来自手表记录的传感器数据 + 联网修正，精确到训练时段。
> 未授权时降级使用 Open-Meteo，参考 `weather-analysis.md`。

## 五、何时优先使用网页端

✅ **必须使用网页端的情况：**
- 晨跑 vs 夜跑判断（影响配速解读和着装建议）
- 同一天多次训练需要区分
- 需要训练时的精确温度和湿度（而非全天均温）
- 间歇训练质量分析需要完整计圈表

✅ **建议使用网页端的情况：**
- DI 不适指数计算需要精确温湿度
- 配速受天气影响的量化分析
- 训练报告需要"时间段"描述

❌ **不需要网页端的场景：**
- 只看汇总数据（跑量/负荷/HRV 趋势）
- 单次训练纯文字诊断（用 MCP `analyzeActivityDetail`）
- 周/月汇总分析

## 六、与 skill 工作流集成

### 在 SKILL.md 中的定位

```
单次训练分析（标准流程）
  ① 调用 getActivityDetail 获取汇总
  ② → 判断是否需要时间/天气精度 ← 新增
  ③ 拉取训练当天天气 → 计算 DI
  ④ 查前3天跑量和睡眠（疲劳因素）
  ⑤ 综合解读
```

### 在首次分析中的定位

```
首次分析用户
  ① 拉取 6 组数据
  ② → 可选：网页端拉取最近2次训练的精确时间/天气 ← 可选细化
  ③ 输出结论
```
