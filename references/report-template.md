# 分析报告模板参考

## 一、输出行为约定

> **默认：对话框直接输出分析结论，不写本地文件。**
>
> 保存文件时不得重新拉取数据和分析，直接复用当前会话已有结论。
> 报告末尾询问用户是否需要保存为 HTML。

### 文件输出流程

1. **核心规则**：不往 C 盘写；不重新拉数据；存 HTML；CSS 内联
2. **默认目录**：`{workspace}/高驰数据分析/`
3. **用户确认**：首次保存时询问是否使用默认目录
4. **自定义路径**：允许用户指定任意绝对路径，存入工作区 memory
5. **目录不存在**：自动创建
6. **文件名**：`运动数据分析报告_YYYYMMDD.html`
7. **CSS 样式**：内联于 `<head><style>`，如下：

```css
body { font-family:-apple-system,'Segoe UI','Noto Sans SC',sans-serif; max-width:900px; margin:0 auto; padding:20px; background:#f8f9fa; color:#333; line-height:1.7; }
h1 { color:#1a1a2e; border-bottom:3px solid #e94560; }
h2 { color:#16213e; border-left:4px solid #0f3460; padding-left:12px; }
table { border-collapse:collapse; width:100%; margin:16px 0; font-size:14px; }
th { background:#0f3460; color:white; padding:10px 12px; }
td { padding:8px 12px; border-bottom:1px solid #dee2e6; }
tr:nth-child(even) { background:#f1f3f5; }
```

## 五、最快单圈分析（口径修正）

⚠️ **"最快单圈"在三个数据源中含义不同，2026-06-08 用户纠正后口径如下：**

| 数据源 | 字段名 | 真实含义 |
|--------|--------|---------|
| **高驰 APP** | 最快单圈 | 所有计圈中**配速最快**的那一圈（被标记"最快"）|
| **高驰 网页端 "概要"** | 最快1公里 | 任意 1km 段的最快配速（**不一定是计圈**）|
| **MCP `getActivityDetail`** | 最快单圈 | 实际 = 网页端的"最快1公里"（**不是**计圈）|

**实证：**
- 5/27「400米间歇8组」：APP 3:45/km，网页端/MCP 4:36/km → 差 51s
- 6/4「间歇800×8」：APP 3:51/km，网页端/MCP 4:16/km → 差 25s

**结论：**
- ❌ 用 MCP"最快单圈"评估间歇训练质量 = 错误指标
- ✅ 间歇训练质量评估：使用 `web-detail-fetcher.md` 抓取网页端计圈表，表中"最快"标记行的配速

**普通训练分析（如慢跑、长距离）口径：**

最快单圈 = 一次训练中所有自动计圈里配速最快的一圈，反映本次训练的上限能力。

| 指标 | 说明 | 价值 |
|------|------|------|
| 均配速 | 整体均值，受慢段拖累 | 综合完成能力 |
| 最快单圈 | 多次计圈中最快配速 | 极限片段，可对比阈值差距 |

最快单圈比均配速快 10-30s/km，差距越小配速越均匀。
