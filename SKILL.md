分析 COROS 高驰手表用户的训练数据、运动表现和恢复状态。

## 三、核心工作流

### 首次分析用户

```
① 并行拉取 6 组数据（fitness/health/records/recovery/load/hrv）
② 提取 fitness 字段（VO2max、预估成绩、阈值配速/心率）
③ 拉取近 K 天天气概览
④ 输出分析结论
```

### 单次训练分析

```
① 调用 getActivityDetail 获取汇总
② 拉取训练当天天气数据 → 计算 DI
③ 查前3天跑量和睡眠（疲劳因素）
④ 综合解读：疲劳 > 天气 > 训练意图
```

## 四、必须记住的关键规则

### 4.1 睡眠日期错位（高驰 API 特性）

`querySleepData` 返回的日期 = **醒来当天**，不是入睡当天。

| API 返回 | 实际入睡 |
|---------|---------|
| M/D | (M/(D-1)) 晚 |
| N 日 | (N-1) 日 |

查"N晚"睡眠 → endDate 设为 N+1。

### 4.2 运动记录无时间戳

`querySportRecords` 的日期字段不含时分秒，记录顺序不保证时间顺序。
- ❌ 不推断上午/下午
- ✅ 同一天多次运动询问用户确认

### 4.3 分析优先级

**疲劳 > 天气 > 训练意图**。
查天气前，先查前3天跑量和睡眠。

### 4.4 保存文件规则

- 默认输出目录：`{workspace}/高驰数据分析/`
- 不往 C 盘写文件
- 不重新拉取数据（直接复用当前会话结论）
- 存 HTML 格式，CSS 内联
- 报告末尾询问用户是否保存

## 六、调试要点

1. 数据疑似不对 → 先确认工具 inputSchema
2. 用 `queryDailyHealthData` 交叉比对（最稳定）
3. 多次请求返回内容完全相同 → 参数名写错
